# Frontend Typecheck Stability

## What Completed In This Slice

This slice makes the frontend type-check command reliable on the first run instead of depending on whatever `.next/types` state happened to exist locally.

Completed outcomes:
- `front/package.json` now routes `npm run type-check` through a dedicated bootstrap script instead of calling `tsc --noEmit` directly.
- `front/scripts/type-check.mjs` checks whether required Next-generated type markers already exist and runs `next build` once when they do not.
- The same wrapper also disables incremental mode for the explicit type-check pass so stale `.next/types` references from a previous graph do not keep breaking clean validations.
- When TypeScript still reports `.next/types`-related `TS6053` errors, the wrapper rebuilds once and retries automatically before surfacing a failure.
- The bootstrap path now clears stale `.next` build artifacts before rebuilding generated route types, which avoids intermittent `ENOTEMPTY` cleanup failures during retry builds.

## Root Cause

The App Router type surface is generated by Next under `.next/types`.

The previous `tsc --noEmit` script had two weak spots:
- on a fresh checkout, or after `.next` was cleaned, the generated route types might not exist yet
- when `tsconfig` incremental metadata still referenced an older `.next/types` graph, `tsc` could fail with missing generated files even though the source tree itself was valid

That made the frontend type-check step flaky and forced repeated runs or a manual `next build` before the result became trustworthy.

## Implementation

The stabilized command now follows this flow:

1. Check whether required generated markers such as `.next/types/package.json` and `.next/types/app/page.ts` exist.
2. If they do not exist, run `next build` once to bootstrap generated types.
3. Run `tsc --noEmit --incremental false` as the actual verification command.
4. If the first pass still reports `.next/types`-related `TS6053` errors, rebuild once and retry the same type-check exactly one time.

## Validation

Validated in this slice:
- `npm run type-check`
- `npm run type-check` again immediately after the first run
- `npm run build`
- `npm run type-check` again after the build
